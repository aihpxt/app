# AI 服务平台软件说明书

## 第一章 软件概述

### 1.1 软件基本信息

| 项目 | 内容 |
|------|------|
| 软件名称 | AI 服务平台 |
| 软件简称 | AI Service |
| 版本号 | V1.0.0 |

| 首次发表日期 | 2026-04-25 |
| 软件类型 | 应用软件 |

| 软件大小 | 约 50 MB |

### 1.2 软件简介

AI 服务平台是一个基于 FastAPI 框架构建的智能服务系统，提供智能体聊天、任务调度、系统监控等功能。系统采用分层架构设计，集成了 SQLite 数据库、Redis 缓存、Hermes 智能体分派系统等核心组件，为用户提供高效、智能的服务体验。

### 1.3 主要功能

AI 服务平台提供以下主要功能：

1. **智能对话系统**：基于 AI Agent 的智能问答，支持多轮对话、意图识别、上下文管理
2. **学校数据管理**：学校信息查询、筛选、详情展示，支持多条件组合查询
3. **政策信息管理**：政策法规查询、分类、搜索，支持全文检索
4. **智能体编排**：Hermes Agent 智能分派，支持多智能体协作
5. **数据采集系统**：OpenClaw 爬虫系统，支持多源数据采集
6. **系统监控**：实时性能监控、告警管理，支持 Prometheus 集成
7. **多级缓存**：L1 内存缓存 + L2 Redis 缓存，支持自动切换
8. **熔断保护**：服务容错保护，支持自动恢复
9. **智能重试**：指数退避重试策略，提高系统可靠性
10. **用户认证**：JWT 认证、RBAC 权限管理

## 第二章 系统架构

### 2.1 整体架构

AI 服务平台采用分层架构设计，包括表现层、业务逻辑层、数据访问层和基础设施层。

```
┌─────────────────────────────────────────────────────────────┐
│                      客户端应用                              │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                    API Gateway                              │
│  ┌─────────────┬─────────────┬─────────────┐               │
│  │ 认证/授权    │  限流保护    │  请求路由   │               │
│  └─────────────┴─────────────┴─────────────┘               │
└─────────────────────────┬───────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ AI Service │   │ Hermes   │   │ OpenClaw │
    │  (主服务)  │   │ Agent    │   │  Crawler │
    └──────────┘   └──────────┘   └──────────┘
          │               │               │
          └───────────────┼───────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ SQLite   │   │  Redis   │   │ 外部API  │
    │ Database │   │  Cache   │   │  数据源   │
    └──────────┘   └──────────┘   └──────────┘
```

### 2.2 架构特点

1. **模块化设计**：各模块独立开发、测试、部署，便于维护和扩展
2. **分层架构**：清晰的分层设计，降低模块间耦合度
3. **服务集成**：支持 Hermes、OpenClaw 等外部服务集成
4. **数据分离**：数据库和缓存分离，提高数据访问效率
5. **监控完善**：完善的监控和告警机制，便于运维管理

### 3.1 智能对话系统

### 3.5 多级缓存系统

#### 3.5.1 功能描述

多级缓存系统实现 L1 内存缓存 + L2 Redis 缓存的自动切换机制，提高系统性能。

#### 3.5.2 缓存架构

```
L1 Cache (内存) ←→ L2 Cache (Redis) ←→ Database
```

#### 3.5.3 主要特性

1. **自动回源**：缓存未命中时自动从下级缓存或数据库获取
2. **双写机制**：写入时同时写入 L1 和 L2
3. **缓存预热**：系统启动时预加载热点数据
4. **TTL 管理**：自动管理缓存过期时间
5. **LRU 淘汰**：内存缓存使用 LRU 淘汰策略

### 3.6 熔断器

#### 3.6.1 功能描述

熔断器模式在服务调用失败时自动熔断，保护系统稳定性。

#### 3.6.2 状态机

```
CLOSED ──→ OPEN ──→ HALF_OPEN ──→ CLOSED
   ↑                          │
   └──────────────────────────┘
```

- **CLOSED**：正常状态，允许请求通过
- **OPEN**：熔断状态，拒绝所有请求
- **HALF_OPEN**：半开状态，允许部分请求通过

### 3.7 智能重试

#### 3.7.1 功能描述

智能重试机制基于指数退避策略，提高系统可靠性。

#### 3.7.2 重试策略

1. **指数退避**：每次重试间隔时间翻倍
2. **最大重试次数**：防止无限重试
3. **超时控制**：单次请求超时控制
4. **异常过滤**：可配置忽略特定异常

## 第四章 技术特点

### 4.1 分布式缓存

系统使用 Redis 作为分布式缓存，具有以下特点：

- **高性能**：Redis 是内存数据库，读写速度极快
- **持久化**：支持 RDB 和 AOF 两种持久化方式
- **分布式**：支持集群部署，提高可用性
- **丰富数据结构**：支持字符串、哈希、列表、集合等多种数据类型

### 4.2 服务监控

系统实现完善的服务监控机制：

- **实时监控**：使用 Prometheus 实时采集指标
- **告警管理**：AlertManager 统一管理告警
- **可视化**：Grafana 展示监控面板
- **自动恢复**：服务异常自动重启

### 4.3 缓存预热

系统实现智能的缓存预热机制：

- **启动预热**：系统启动时加载热点数据
- **动态预热**：根据访问热度动态调整缓存
- **后台刷新**：定期刷新即将过期的缓存

### 4.4 连接池管理

系统使用连接池优化数据库和 Redis 连接：

- **复用连接**：减少连接创建销毁开销
- **连接限制**：防止连接泄漏
- **健康检查**：自动检测无效连接

### 4.5 异步处理

系统使用 asyncio 实现异步处理：

- **非阻塞I/O**：提高并发处理能力
- **并发请求**：同时处理多个请求
- **资源优化**：减少线程开销
