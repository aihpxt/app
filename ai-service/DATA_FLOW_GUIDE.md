## 2. 典型对话流程详解

### 2.1 场景：用户询问"它的分数线是多少"

#### 第一阶段：请求接收
```
用户输入: "它的分数线是多少"
     │
     ▼
API路由层 (ai.py)
     │
     ├─ 1. 解析请求参数
     │      session_id: "abc123"
     │      message: "它的分数线是多少"
     │
     ├─ 2. 生成会话ID (首次对话)
     │      session_id = hash("它的分数线是多少" + timestamp)
     │
     └─ 3. 返回 session_id 给前端
```

#### 第二阶段：历史加载
```
API路由层
     │
     ▼
UnifiedDatabaseManager
     │
     ├─ SELECT * FROM chat_messages
     │   WHERE session_id = "abc123"
     │   ORDER BY created_at ASC
     │
     ▼
返回历史消息:
[
  {"role": "user", "content": "我想了解师大附中"},
  {"role": "assistant", "content": "云南师范大学附属中学..."}
]
```

#### 第三阶段：上下文处理
```
历史消息 + 当前输入
     │
     ▼
指代词解析器 (PronounResolver)
     │
     ├─ 检测到指代词: "它"
     ├─ 从历史中提取: "师大附中"
     ├─ 解析结果: ResolvedEntity(
     │      entity_type="school",
     │      entity_name="师大附中",
     │      confidence=0.95
     │  )
     └─ 替换后的输入: "'师大附中'的分数线是多少"
```

#### 第四阶段：智能体处理
```
替换后的输入 + 上下文
     │
     ▼
ControlCenterSpecialistAgent.handle()
     │
     ├─ 1. 上下文构建
     │      context_school = "师大附中"
     │      topic = "昆明学校"
     │      conversation_history = [...]
     │
     ├─ 2. 规则匹配
     │      检测到:
     │      - 指代词 ✓
     │      - 分数线关键词 ✓
     │      - context_school ✓
     │
     ├─ 3. 响应生成
     │      response = """
     │      根据您提到的"师大附中"，我来为您查询相关信息。
     │      ...
     │      """
     │
     └─ 4. 会话状态更新
             topic = "昆明学校"
             conversation_history.append(...)
```

#### 第五阶段：Hermes增强（可选）
```
基础响应 + 会话上下文
     │
     ▼
Hermes服务 (如果可用)
     │
     ├─ 1. 情感分析
     │      emotion = "期待"
     │      sentiment = "积极"
     │
     ├─ 2. 意图识别
     │      intent = "information_inquiry"
     │      entities = ["师大附中", "分数线"]
     │
     ├─ 3. 洞察生成
     │      missing_info = ["考生分数", "所在地区"]
     │      followup_topic = "请问您孩子的中考成绩是多少"
     │
     └─ 4. 个性化增强
             添加跟进问题，调整语气
```

#### 第六阶段：数据保存
```
最终响应 + 会话状态
     │
     ▼
并行执行:
     │
     ├─► 保存到 SQLite
     │      INSERT INTO chat_messages
     │      (session_id, role, content, created_at)
     │      VALUES
     │      ('abc123', 'user', '它的分数线是多少', '2024-01-15T10:30:00'),
     │      ('abc123', 'assistant', '根据您提到的...', '2024-01-15T10:30:01')
     │
     └─► 保存到 Redis
             key: "session:abc123"
             value: {
                 "topic": "昆明学校",
                 "conversation_history": [...],
                 "last_question": "它的分数线是多少",
                 "last_answer": "根据您提到的...",
                 "timestamp": 1705295401
             }
             TTL: 7200
```

#### 第七阶段：响应返回
```
最终响应
     │
     ▼
API路由层
     │
     └─ 返回给前端
            {
                "success": true,
                "data": {
                    "content": "根据您提到的...",
                    "session_id": "abc123"
                }
            }
```

## 6. 错误处理与降级策略

### 6.1 分层降级机制
```
┌─────────────────────────────────────────────────────────────┐
│                    第一层: 正常模式                          │
│                                                             │
│  用户输入 → 智能体 → Hermes增强 → 返回                      │
│                     │                                        │
│                     ▼ (Hermes不可用)                        │
├─────────────────────────────────────────────────────────────┤
│                    第二层: 降级模式                          │
│                                                             │
│  用户输入 → 智能体 → (跳过Hermes) → 返回                    │
│                     │                                        │
│                     ▼ (智能体失败)                           │
├─────────────────────────────────────────────────────────────┤
│                    第三层: 备用模式                          │
│                                                             │
│  用户输入 → 规则引擎 → FAQ匹配 → 返回                       │
│                     │                                        │
│                     ▼ (FAQ无匹配)                            │
├─────────────────────────────────────────────────────────────┤
│                    第四层: 兜底模式                          │
│                                                             │
│  用户输入 → "抱歉，我无法理解您的问题，请重新描述"           │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 各层超时配置
```
智能体处理:  5秒超时
Hermes服务:  2秒超时 (可选)
数据库查询:  3秒超时
Redis缓存:   1秒超时
```

### 6.3 错误日志记录
```python
try:
    response = agent.handle(user_input, context)
except AgentException as e:
    logger.error(f"智能体处理失败: {e}")
    # 降级到规则引擎
    response = rule_engine.handle(user_input, context)
except HermesException as e:
    logger.warning(f"Hermes服务不可用: {e}")
    # 跳过Hermes增强
    response = base_response
except DatabaseException as e:
    logger.error(f"数据库错误: {e}")
    # 使用缓存数据或返回错误
    raise ServiceUnavailableException("服务暂时不可用")
```

## 7. 性能优化要点

### 7.1 数据库优化
- 为 `session_id` 和 `created_at` 创建索引
- 定期清理过期会话数据
- 使用连接池复用连接

### 7.2 缓存优化
- 热点数据预加载到Redis
- 会话状态异步保存
- 使用Pipeline批量操作

### 7.3 并发优化
- 使用异步IO处理请求
- 并行执行数据库和缓存操作
- 限制并发请求数量

### 7.4 响应优化
- 预加载常见查询结果
- 使用模板缓存
- 启用HTTP压缩
